User Transition Examples
Scenario 1: From Basic to Intermediate
User Context:
Sarah (QA Manager) has been using the basic Test Execution Performance widget for 2 weeks. She successfully created her first widget showing "Average test duration over last 30 days" and added it to the team dashboard.
Transition Trigger: Sarah notices execution times are higher on Fridays and wants to compare with previous months to see if this is a pattern.
Progressive Disclosure Response:
System shows "Show More Options" with a hint: "Need to compare with previous periods or add more filters?" When clicked, reveals time comparison dropdown and environment filters - exactly what Sarah needs, without overwhelming complexity.
Scenario 2: From Intermediate to Advanced
User Context:
Mike (Senior QA Lead) has created multiple widgets with custom filters and time comparisons. He's comfortable with the intermediate interface and has built 5 different performance analysis widgets for various team needs.
Transition Trigger: Mike wants to create a "Performance Score" combining execution time, pass rate, and team velocity into a single metric for executive reporting.
Progressive Disclosure Response:
System detects Mike's usage pattern and proactively suggests "Advanced Settings" with calculated fields capability. The interface reveals a formula builder with business-friendly functions, examples, and validation - bridging the gap to advanced capabilities.
Scenario 3: From Advanced to Expert
User Context:
Lisa (QA Director & Analytics Champion) has mastered advanced configuration and created complex cross-entity analysis widgets. She's now the go-to person for team analytics and wants to optimize query performance and create custom datasets.
Transition Trigger: Lisa needs to create a custom dataset combining data from external sources and wants to optimize query performance for company-wide dashboards.
Progressive Disclosure Response:
System recognizes Lisa as a power user and offers "Expert Mode" access. This unlocks direct SQL editing, custom dataset creation, and performance optimization tools - essentially providing controlled access to Superset's full capabilities within the Katalon interface.
